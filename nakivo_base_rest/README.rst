Reusable REST primitives for Odoo addons.

Provides:

- request payload parsing and Pydantic-based validation
- a centralized exception hierarchy mapped to stable application error
  codes
- a single response envelope shared by every consumer addon
- helpers to enforce authentication and group membership inside
  ``type='http'`` controllers returning JSON responses

Usage
=====

Inherit ``BaseRestHandler`` in your controller and call
``_dispatch_api`` to wrap the request lifecycle. Use
``_validate_payload`` with a Pydantic schema to validate incoming data,
and ``_make_success_response`` / ``_make_error_response`` to return the
standard envelope.

The handler maps Odoo ``UserError``, ``ValidationError``,
``AccessError``, ``MissingError``, and Pydantic ``ValidationError``
instances into stable API exceptions automatically.
